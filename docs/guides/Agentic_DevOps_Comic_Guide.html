The Agentic Adventure

How your app ships to the world & how your AI agent actually thinks — explained in panels.

ADA · your guideHi! I'm Ada. I'll walk you through deploying apps and building AI agents — no prior DevOps required. By the end you'll explain these to anyone.

Ch.1 — A Tale of Two Tiers

🌍 The front of the shop

Your frontend is the React app. When you run yarn build it becomes plain HTML/JS/CSS — just files. Files are easy: a CDN copies them to servers worldwide so they load fast. That's Netlify's job.

🧠 The workshop in the back

Your agent is different. It runs a real language model (Ollama) that stays loaded in memory and thinks for several seconds. That needs a long-running machine — your computer or a server — wrapped in Docker.

YOUSo why can't Netlify just run the agent too?
ADABecause Netlify runs serverless functions — tiny code that wakes up, runs ~10 seconds, and vanishes. No room to keep a big model loaded. Perfect for a quick API call, wrong for a thinking agent.

🗺️ The whole map

Docker is the shipping box 📦 around the back tier: it packs the API, the model, and the database so they run identically on your laptop and your server. One command: docker compose up.

Ch.2 — Shipping It (DevOps decoded)

🔁 The magic words: CI / CD

CI = every time you change code, a robot builds it and runs the tests. CD = if the tests pass, it goes live automatically. Together: you push code, and minutes later it's on the internet — safely.

git push→ CI: build + test→ Preview URL→ Live!

🚀 Your standard deploy flow

  1. Connect the GitHub repo to Netlify (once).
  2. Open a Pull Request → Netlify gives you a preview URL to check it.
  3. CI must go green (lint + tests) before merge.
  4. Merge to main → it auto-deploys to production. 🎉
  5. Something broke? One-click rollback to the last good version.

🛡️ Two more characters you'll meet

Reverse proxy — the bouncer at the door (Nginx/Caddy). It adds HTTPS 🔒 and sends visitors to the right room.
Environment variables — secret sticky-notes (passwords, DB links) handed to the app at runtime, never written into the code.

Ch.3 — How an Agent Thinks

🤖 Not a chatbot — a doer

A normal chatbot just talks. An agent can act: it has tools (like "create task" or "log expense") and decides when to use them. The decision cycle is the agent loop:

🧠 Think→ 🔧 Use a tool→ 👀 See result→ ↻ Repeat / Answer
YOU"Log ₹400 lunch and remind me to call Rohan."
ADAI split that into two actions → call log_expense and create_task → check both worked → reply "Done!". That's the loop in real life.

🧰 Why tools are the safe way

The model never touches your database directly. It can only ask to run a tool, and each tool validates the request and checks it's really you. That's how you keep an AI powerful and safe — and it's why adding a new ability = just writing a new tool.

Ch.4 — Memory & RAG

📚 The meaning-map (embeddings)

Computers don't read words — they read numbers. An embedding turns a sentence into a point on a giant "map of meaning". Sentences that mean similar things land close together, even with different words.

🔎 RAG = look it up first

RAG (Retrieval-Augmented Generation) means: before answering about your life, the agent searches your real notes/goals/captures (using the meaning-map), grabs the closest ones, and answers from those. Result: grounded answers with sources — not made-up ones.

❓ Your question→ 🔎 Vector search→ 📄 Your real notes→ ✅ Grounded answer

In your project these live in MongoDB's embeddings collection — open Compass and you can literally see them. Full control. 🔬
